## Scoring Comparison

### LOCOMO
- LLM Judge: 1 (correct) or 0 (wrong)
- Simple average across question types
- No penalty for hallucination

### LongMemEval
- Recall@5 + Accuracy
- LLM Judge (GPT-4o or Gemini)
- 5 memory abilities tested separately
- No penalty for hallucination

### WMB-100K
- LLM Judge: CORRECT (0.1 pts) or WRONG (0 pts)
- **False memory penalty: -0.25 per hallucination**
- **Two-part scoring:**
  - Part A: Pre-trained Knowledge (100 pts)
  - Part B: Conversation Memory (100 pts)
  - Total: 200 pts max
- **5 difficulty levels (L1-L5) reported separately**
- **Speed measured separately for Part A and Part B**
- Scoring cost: ~$0.07 (transparent)

## Why WMB-100K Is Harder

### 1. Scale Breaks Naive Approaches
At 600 turns, you can fit everything in a single context window. WMB-100K tests 20 sections independently — each section is ~200-230K tokens. While a 1M context model could handle individual sections, it must process all 20 sections separately. Memory systems that rely on context stuffing must pay full LLM costs per section with no cross-section recall.

### 2. False Memory Penalty
LOCOMO and LongMemEval don't test for hallucination. A system that confidently returns wrong answers gets the same score as one that says "I don't know." WMB-100K penalizes false memories at **2.5x** the reward for correct answers. A system that hallucinates on 400 probes loses 100 points — enough to fail completely.

### 3. Two-Part Testing
Other benchmarks only test conversation memory. WMB-100K also tests retrieval of pre-loaded documents (Part A), simulating enterprise use cases where an AI needs to remember both documentation AND conversations.

### 4. L2-L5 Multi-Search Questions
L1 (single fact lookup) is what existing benchmarks test. WMB-100K adds:
- **L2**: Find 2 related facts and connect them
- **L3**: Time-aware cross-referencing
- **L4**: Track fact changes over time (update chains)
- **L5**: Synthesize 3+ facts from different time periods

Single-vector-search systems (like basic RAG) fail at L2+.

### 5. Natural Conversation Noise
100,000 turns of natural dialogue (generated by Claude Haiku) bury facts in realistic conversation. Facts at turn 40 are covered by 99,960 turns of noise. No keyword matching can survive this.

## False Memory Defense

No existing benchmark tests for hallucinated memories. WMB-100K is the first:

```
FM Defense Rate (higher = better)

  Mem0          ██ 1.8%  (393/400 hallucinated)
  LangChain     ▏ 0.0%  (400/400 hallucinated)
```

Both systems confidently return fabricated information for questions about things never mentioned.

### Why Full Context Is Not Viable

| Model | Cost (3,224 questions) |
|-------|----------------------|
| GPT-4o-mini | $98 |
| GPT-4o | $1,638 |
| Claude Sonnet 4.6 | $1,967 |
| Claude Opus 4.6 | $9,835 |

Full context requires loading ~230K tokens per section for each question. At 3,224 questions across 20 sections, the input token cost alone makes it impractical for production use.
